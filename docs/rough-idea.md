I want to make an AI RPG game engine. It's kind of a framework for building text based RPG games. The games are going to be primarily text based. This means scenes, characters, dialogues and actions are going to be text based. Images and videos can be embedded. Background music and audio effects can also be embedded. This is going to be like common 2D novels, but with the text based interactivity.

What I'm writing right now is like a rough idea. I'm just pouring my thoughts. The entire game will be generated using AI. The tool we are building is just an engine. The user will give the initial prompt and directions to the engine. And the engine will start the gameplay. For the initial basic version, I want to skip all the visual elements. Let's start with 100% text version.

Now I have to think from the perspective of the user. That's how I can bring clarity to my idea. Suppose I, as a user, want to develop a game. I don't know exactly what game I'm going to make, but I have a picture of "Stranger Things". In that they are playing some kind of game. And they play in terms of roles. There are locations. There are characters. There are dialogues and actions of characters. Players have the options to make choices. Their choices affect the storyline. There are also some stats associated with characters and places and stuff. Choices affect those stats. Higher and lower values of those stats open new possibilities. There is some randomness and unpredictability also. But I don't want to limit this as a game of stranger things. I want to develop this project as an engine for such a game. You can have whatever storyline in your mind. You can define the complete setting of the story. You can define the details of the characters. Details of the characters means: Who the character is? How he look like? What he thinks? How he make decisions? You can configure this only in the beginning. Once you commit to the game, you cannot manipulate them later on. You have to progress the storyline.

This sounds like a very big idea. But I want to simplify it and develop something minimum. Once I have something minimum in my hand, I can see it working. Then I can decide the direction of the project.

Let's start with the minimum. Let's develop the basic mechanism of creating a story and playing it.

For now I am thinking about the following objects:

1. Location: It's a place where a scene happens. It can represent anything from a jungle to a royal courtyard to a room or whatever. A location can have some items or characters present in it. Of course, the presence of characters at a location is dynamic. You cannot just define that a character is always present, at some location.

2. Character: It's exactly what it says. We don't define characters have what properties. This is completely dependent on the user. What kind of characters they want to develop? What kind of properties and traits the character would have? Characters can have some stats. These stats will be changed based on the choices. User can define the rules and behavior of these stats. User can define the range of these stats and what will make them increase and what will make them degrees. User can define what new opportunities and possibilities will open given the stat will reach some value.

3. Scene: It's the detail of the current chapter. Like, what exactly is happening in the story? This is the primary source for AI to know about the story. AI will try to continue the scene. That's the whole point of this RPG.

Now comes the question: How exactly is the user is going to input all such information? User will use the schema less JSON. This means that there is no proper decided structure of the JSON. The JSON is only a language to structurally define everything. There is no expected structure of the JSON that the user is going to input. The user inputted JSON will be provided as a context to the AI.

Even the entire process of writing such a JSON would be automatically done by the user using some other AI. But that's not the job of this project. The user will tell his story to like ChatGPT. And ask it to generate the complete JSON that can be pasted in the project. The project is nothing but the game engine. It will provide the JSON to the AI.

The job of the AI: Generate the dialogues of the characters and actions of the characters. OK here comes the most important thing: A story can have, of course, many characters. But the user will be playing the role of only one character. The role of all other characters will be played by the AI. The user have no control over the dialogues and actions and choices of other characters. The user can only decide the dialogues and choices of his own character.

The game AI is given a proper prompt. In that prompt, it is instructed to generate a structured Json. That structured Json is very strictly to be followed. This will contain the dialogues and actions of the user's character. That's precisely the exact job of AI in this game. The AI is going to present three options to the user. Every option has some orientation and cost. Like one option can lead to mysterious outcome. Another option can lead to horror. Another item can lead to victory or another item can lead to defeat. Some options will increase some stats. Some options will decrease some stats. So I am like every choice has its own impact and direction. These choices will be presented in front of the user and user will make a choice. After the user make a choice, the story would continue. So it's like a loop. In every turn user makes a choice and the story goes forward. Based on the user's made choices, the other characters react and the story goes on. Technically, the game never ends, actually. It can just go on forever.

We are going to use Vue js for this. We are not going to use any state management library. We're going to use simple reactive stores. We're going to use tailwind for styling. We are not going to use Typescript. The style is going to be very minimal. Use CSS only wherever necessary. Write clean and minimal code. Spread the logic in components. I like modular code. We are going to use Google Gemini AI. I will paste the API key in the environment file. We are going to use I think maybe some kind of SDK. And we will use some kind of chat mode for AI. This is essentially a chat actually. In users term you specify the exact choices the user made. Then the in the next response system will tell the upcoming choices. Also updating the stats as per the user's made choices.

For the very simplicity, let's assume it's a simple role play between two people and have some location. Maybe they're in a house. And both people have some stats. The user is going to play the role of one of the person. User has no control over the choices of the other person. Both people have some stats. Those stats are going to be affected by user's made choices.
